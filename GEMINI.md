# Gemini CLI - choichoi Project Context

Welcome to the `choichoi` project. This document provides the essential context, architecture, and development guidelines for this repository.

## Project Overview

`choichoi` is a specialized **Point of Sale (POS)** and event management application designed for popup stores and small retail events. It provides a keyboard-optimized interface for quick order processing, along with administrative tools for menu management, sales tracking, staff scheduling, and internal memos.

### Core Tech Stack

- **Framework:** [Next.js 16 (App Router)](https://nextjs.org/)
- **UI Library:** [React 19](https://react.dev/)
- **State Management & Data Fetching:** [TanStack React Query (v5)](https://tanstack.com/query/latest)
- **Backend & Database:** [Supabase](https://supabase.com/) (PostgreSQL)
- **Styling:** [Tailwind CSS v4](https://tailwindcss.com/)
- **Package Manager:** [Yarn](https://yarnpkg.com/)
- **Deployment:** [Vercel](https://vercel.com/)

## Architecture & Data Flow

The project follows a modern Next.js architecture with a clear separation between client-side interactivity and server-side data persistence.

### Key Components

- **POS Interface (`app/page.js`):** The heart of the application. It features keyboard shortcuts (1-9 for menu items, Enter for checkout, Esc for reset) for high-speed operation during events.
- **Server Actions (`app/actions.ts`):** Bridges the client components with the backend. It wraps Supabase utility functions and provides a consistent interface for data mutations.
- **Database Layer (`lib/supabase.js`):** Contains the direct logic for interacting with Supabase tables (`menu_items`, `orders`, `order_items`, `popup_events`, `schedule_slots`, `memos`).
- **Access Control (`app/password-gate.js`):** A client-side overlay that requires a password for accessing the operational screens, stored in `localStorage`.

### Routing Structure

- `/`: POS (Point of Sale) terminal.
- `/stats`: Sales statistics, monthly calendars, and item-by-item revenue breakdowns.
- `/schedule`: Staff scheduling and popup event management.
- `/memo`: Internal notes and memos system.
- `/settings`: Menu configuration (CRUD, drag-and-drop reordering) and database maintenance.

## Development Guidelines

### Coding Standards

- **Language:** The project uses a mix of JavaScript (`.js`, `.jsx`) and TypeScript (`.ts`). New logic should prefer TypeScript where possible.
- **Data Fetching:** Use **React Query** for client-side data fetching and synchronization to ensure the UI stays updated without manual refreshes.
- **State Updates:** Use **Server Actions** for all data mutations (POST, PUT, DELETE) to leverage Next.js's built-in caching and security features.
- **Styling:** Adhere to the established CSS patterns in `app/globals.css`. Prefer Tailwind utility classes for layout and spacing.

### Keyboard Optimization

When modifying the POS interface (`app/page.js`), ensure that keyboard accessibility and shortcuts are maintained. The system is designed to be usable without a mouse.
